    会议控制API说明汇总

    会议控制API说明汇总

    一、常量

    会议的分屏模式

    分屏模式的值列举如下： auto 1X1，1X2，2X1，2X1H，1X2V， 1TP2H，1BP2H，1LP2V， 2X2，1BP3H，1LP3V，1TP3H， 1LP4V，1BP4H，1TP4H， 1P5，2X3， 1P7， 3X3，1MP8，1BP8，1TP8， 2MP8，1CP12，4X4，1P16，1CP16，4X5，4X5S， 1P20，1CP20，5X5，5X6，5X6S，6X6

    说明： T表示Top，B表示Bottom，C表示Center，M表示Middle，L表示Left，H表示Horizontal，V表示Vertical，P表示Plus，S表示Square

    会议状态

    取值（类型为String） 说明
    APPROVED 预约成功
    ONGOING 正在进行中
    FINISHED 已结束
    FAILED 失败
    DELETED 已被删除

    终端状态

    取值（类型为String） 说明
    OFFLINE 离线
    ONLINE 在线
    INCALL 呼叫中
    UNKNOWN 未知

    终端类型

    取值（类型为String） 说明
    M16 -
    M18 -
    VE/VL -
    HEXMEET 软终端
    OTHER 其他类型

    呼叫协议

    取值（类型为String） 说明
    SVC -
    SIP -
    H.323 -

    双流分屏

    取值（类型为String） 说明
    oneByOne 平衡
    contentOnly 仅双流
    largeContentSmallPeople 双流优先
    largePeopleSmallContent people流优先
    peopleOnly 仅people流

    录制/直播类型

    取值（类型为String） 说明
    GW_WEBCAST 高清
    GW_FHDWEBCAST 超清

    录制状态

    取值（类型为String） 说明
    started 录制中
    paused 已暂停录制
    resumed 已恢复录制
    stopped 已停止录制

    直播状态

    取值（类型为String） 说明
    started 直播中
    stopped 已停止直播

    会议室的容量

    即会议室能同时容许多少终端入会，容量值列举如下：

    10, 25, 50, 100, 200, 400, 1000.

    二、消息结构体定义

    RestConfReq

    简要描述：

    • 预约会议请求体

    参数说明:

    字段 类型 必选 默认 说明
    name String 是 - 会议名称,长度不可超过128个字节
    startTime long 是 - 会议的开始时刻（以毫秒为单位的Unix时间戳），0表示该会议立即开始。
    duration long 是 - 会议的持续时长（单位为毫秒）,建议值为3600000，即一个小时
    lecturerEpId long 否 - 被设置为主会场的某个终端的id
    lecturerUserId long 否 - 被设置为主会场的某个用户的id
    layout String 是 - 取值见常量：会议的分屏模式
    profile String 是 - 会议质量，"SVC"表示质量优先，"SVC_ECO"表示带宽优先，建议值为"SVC".
    password String 否 - 会议密码
    description String 否 - 会议备注
    endpointIds List(Long) 否 - 与会终端id的列表
    userIds List(Long) 否 - 与会用户id的列表
    roomId long 否 - 指定的云会议室的id（适用于云会议室模式）
    maxPartyCount int 是 - 最大入会方数
    useRandomNumericId boolean 是 false 使用随机号码作为会议号码，端口模式下必须设为true
    redialingEnabled boolean 否 false 自动重拨
    recordingEnabled boolean 否 false 录制
    recordingProfile String 否 - 取值见常量：录制/直播类型,建议值为"GW_WEBCAST".
    recordingLayout String 是 - 取值见常量：双流分屏，建议值为"oneByOne".
    liveStreamingUrl String 否 - 推流地址
    adminUserId long 否 - 备选会控者的id
    subtitle RestSubtitle 是 - 字幕
    autoMutingEnabled boolean 否 false 开启自动静音
    allowAnonymousParty boolean 否 false 允许匿名用户入会

    RestSubtitle

    简要描述：

    • 会议字幕

    参数说明:

    字段 类型 说明 必选 备注
    on boolean 启用字幕 是 -
    content String 字幕内容 是 -
    displaySpeed int 滚动速度 是 建议取值：0表示静止，2表示慢速，5表示中速，8表示快速。
    displayRepetitions int 滚动次数 是 滚动速度大于0时才生效.最小值为1,最大值为20,建议值为20.
    verticalBorder int 垂直位置 是 最小值为0，最大值为100，建议值为50.
    transparency int 透明度 是 最小值为0，最大值为100，建议值为50.
    fontSize int 字体大小 是 最小值为9，最大值为32，建议值为24.
    foregroundColor String 前景色 是 十六进制颜色码，建议值为"#FFFF00".
    backgroundColor String 背景色 是 十六进制颜色码，建议值为"#3300CC".

    status String 取值见页面：会议状态

    lecturerUserId long 被设置为主会场的某个用户的id
    layout String 取值见页面：会议的分屏模式
    profile String 会议质量，"SVC"表示质量优先，"SVC_ECO"表示带宽优先.
    password String 会议密码
    description String 会议备注

    subtitle RestSubtitle 字幕
    inCallPartyCount int 当前已成功连接到会议的与会者的数量
    version int 此条会议数据的版本号
    autoMutingEnabled boolean 开启自动静音
    allowAnonymousParty boolean 允许匿名用户入会

    简要描述：

    • 窗口

    参数说明:

    字段 类型 必选 默认 说明

    举例说明 例1：

    {
   "index" : "1",
   "fixed" : false,
   "partyIds" : []
}

    如上所示，fixed取值为false,deviceIds取值为空列表，表示该窗口的显示设置为自动。

    例2：

    {
   "index" : "2",
   "fixed" : true,
   "partyIds": [
               {
                   "deviceId": "425328898",

    三、会议控制API说明

    (1) 会议控制

    简要描述：

    • 用于会议控制

    请求URL：

    • /api/rest/v2.0/ongoingConferences/{confId}/control/{controlType}

    请求方式：

    • PUT

    url中的参数：

    参数名称 详细描述
    confId 会议ID
    controlType 会议控制操作的类型

    请求参数：

    参数名称 是否必选 详细描述
    token 是 登录令牌，所有会议控制操作都需要此参数

    各种会控操作所需的请求参数和请求体：

    controlType的取值 说明 请求参数 请求参数的说明 请求体 请求体的说明
    terminate 结束会议 - - - -
    startLiveStreaming 开启直播 - - RestLiveStreamingReq -
    stopLiveStreaming 停止直播 - - - -
    setLivingLayout 直播分屏 peoplePlusContentLayoutType 取值见常量：双流分屏
    startRecording 开启录制 - - RestRecordingReq -
    pauseRecording 暂停录制 - - - -
    resumeRecording 恢复录制 - - - -
    stopRecording 停止录制 - - - -
    setRecordingLayout 录制分屏 peoplePlusContentLayoutType 取值见常量：双流分屏 - -
    loopPartiesInPanes 分屏轮询 loopPartiesInPanes 取值为”true”表示启用轮询，取值为”false”反之。 RestLoopPartiesInPanes -
    loopBroadcast 广播轮询 loopBroadcast 取值为”true”表示启用广播轮询，取值为”false”反之。 RestLoopBroadcast 当参数loopBroadcast取值为”false”时，请求体应设为空。
    setSubtitle 设置字幕 - - RestSubtitle -
    muteAudioAllParties 全部静音/全部解除静音 muteAudio 取值为”true”表示静音，取值为”false”表示解除静音。 - -
    muteAudioMultipleParties 批量静音/批量解除静音 muteAudio 取值为”true”表示静音，取值为”false”表示解除静音。 List<String> 要操作的与会者的guid列表
    enableRedialing 开启/取消自动重拨 enableRedialing 取值为”true”表示开启自动重拨，取值为”false”表示取消自动重拨。 - -
    setRecordingLivingPeopleLayout 主流分屏 - - RestPartyLayout -
    prolong 会议延时 additionalMinutes 延长的分钟数，最小值为10. - -

    请求Header:

    Content-Type: application/json;charset=UTF-8

    成功返回体说明： 默认值为

    {
    "result":"ok"
}

    特殊情形见下表：

    controlType的取值 说明 返回体 返回体的说明
    muteAudioAllParties 全部静音/全部解除静音 RestPartiesActionResult -
    muteAudioMultipleParties 批量静音/批量解除静音 RestPartiesActionResult -
    prolong 会议延时 RestProlongConfResp -
