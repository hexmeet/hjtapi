    文件管理API

    文件管理API

    一、常量

    文件类型

    取值（类型为String） 说明
    AVATAR 用户头像
    ORG_LOGO 公司LOGO
    CONF_RECORD 会议的录制文件
    WHITEBOARD_FILE 会议的白板文件

    文件链接的用途

    取值（类型为String） 说明
    PLAYBACK 播放
    DOWNLOAD 下载
    COPY_URL 复制链接

    二、消息结构体定义

    RestFile

    简要描述：

    • 文件信息

    参数说明:

    字段 类型 说明 备注
    id long ID -
    fileType String 文件类型 取值见常量：文件类型
    creatorUserId long 文件的创建者的ID -
    creatorUserDisplayName String 文件的创建者的昵称 -
    createMillis long 该文件的创建时间，以毫秒为单位的Unix时间戳 -
    fileSizeInByte long 该文件的字节数 -
    confName String 会议名称 -
    confNumericId String 会议号 -
    downloadCount int 该文件的下载次数 -
    description String 备注 -

    RestFileUrl

    简要描述：

    • 文件的下载链接

    参数说明:

    字段 类型 说明 备注
    fileId long 文件的ID -
    url String 文件的下载链接 -
    use String 该文件下载链接的用途 取值见常量：文件链接的用途

    RestFileDescription

    简要描述：

    • 文件的备注

    参数说明:

    字段 类型 说明 备注
    fileId long 文件的ID -
    description String 文件的备注 -

    RestFileFetchCodeValidationReq

    简要描述：

    • 验证文件提取码的请求

    参数说明:

    字段 类型 必选 默认 说明
    fileUrl String 是 - 通过“复制链接”操作得到的文件链接
    fetchCode String 是 - 通过“复制链接”操作得到的文件提取码

    RestFileFetchCodeValidationResult

    简要描述：

    • 验证文件提取码的结果

    参数说明:

    字段 类型 说明 备注
    successful boolean 验证是否成功 -
    fileUrl String 文件的下载链接 -

    三、文件管理API说明

    (1) 获取会议文件列表

    简要描述：

    • 用于获取会议文件，支持条件查询

    请求URL：

    • /api/rest/v2.0/files

    请求方式：

    • GET

    请求参数：

    参数名称 是否必选 说明 取值
    token 是 登录令牌 -
    fileType 是 文件类型 取值见常量：文件类型
    keyword 否 按会议名称、会议号码或会议的guid搜索时所用的关键词 -

    成功返回体说明： List<RestFile>

    (2) 获取文件的链接

    简要描述：

    • 用于获取文件的链接

    请求URL：

    • /api/rest/v2.0/files/{fileId}

    请求方式：

    • GET

    请求参数：

    参数名称 是否必选 说明 取值
    token 是 登录令牌 -
    use 是 文件链接的用途 取值见常量：文件链接的用途

    成功返回体说明： RestFileUrl

    (3) 填写文件的备注

    简要描述：

    • 用于填写文件的备注

    请求URL：

    • /api/rest/v2.0/files/{fileId}

    请求方式：

    • PUT

    请求参数：

    参数名称 是否必选 详细描述
    token 是 登录令牌

    请求Header：

    Content-Type: application/json;charset=UTF-8

    请求体说明： RestFileDescription

    成功返回体说明： RestFileDescription

    (4) 删除文件

    简要描述：

    • 用于删除文件

    请求URL：

    • /api/rest/v2.0/files/{fileId}

    请求方式：

    • DELETE

    请求参数：

    参数名称 是否必选 说明 取值
    token 是 登录令牌 -

    成功返回示例:

     {
     "result" : "ok"
 }

    (5) 验证文件提取码

    简要描述：

    • 用于验证文件提取码

    请求URL：

    • /api/rest/v2.0/fileFetchCodeValidation

    请求方式：

    • PUT

    请求Header：

    Content-Type: application/json;charset=UTF-8

    请求体说明： RestFileFetchCodeValidationReq

    成功返回体说明： RestFileFetchCodeValidationResult
